# 一站式中实仪审稿

审核中实仪新媒体脚本，确保内容从「顾问式销售思维」而非「媒体思维」出发。

核心判断标准只有一条：
> **客户看完后会觉得你在帮他，还是在教育他？**
> 后者→立场偏了。前者→合格。

中实仪核心业务：**一站式实验室仪器配置**。审稿时必须确保内容链路完整闭合：
> 客户需求 → 检测痛点 → 仪器配置问题 → 配置方案（带场景约束）

---

## 审稿流程

### 第一步：客户画像检查

脚本内容必须能回答三个问题：
- **写给谁？** → 行业 + 岗位（如「化工厂环保负责人」）
- **他在做什么项目？** → 具体场景（如「含汞废水排放验收」）
- **什么让他焦虑？** → 具体痛点（如「验收数据被评审打回」）

判定：如果一个选题「电镀厂/化工厂/电子厂都能看」，画像是太泛。

### 第二步：立场检查（四种错误立场）

**核心原则：不要站在问题的对面，要站在客户的旁边。**
更深一层：要和客户站在同一个战壕里——你在说「我理解你的处境」，而不是「你应该怎么样」。

逐项检查：

#### 1. 监管立场
特征：替环保局/评审老师说话，在吓唬客户。

❌ 企业如果不做汞检测，可能面临处罚
✅ 很多企业在废渣处置时，最大的成本差异往往来自危险特性判定，腐蚀性检测就是重要依据之一
→ 重点变成帮客户降本控险，不是告诉客户会被罚。

#### 2. 专家立场
特征：在证明自己懂，而不是在帮客户解决问题。

❌ 大部分企业都不知道这个检测要求 / 很多实验室都做错了
✅ 在实际项目中，这个环节经常被忽略，容易导致后续复检或返工
→ 前者在评价人，后者在描述问题。

#### 3. 供应商立场
特征：潜意识里想卖设备，忽略了客户真实约束。

❌ 建议配置ICP-MS / 必须配备某某仪器
✅ 如果目前主要检测重金属且样品量不大，可以优先考虑XX方案，后续根据业务量扩展
→ 客户会觉得你在帮他省钱，而不是想卖他设备。

#### 4. 行业批判者立场
特征：把企业当问题，而不是把问题当问题。环保类内容尤其容易踩。

❌ 许多企业为了节约成本直接排放 / 企业环保意识薄弱
✅ 含汞废水来源复杂，部分企业在生产过程中并不容易及时发现风险点
→ 前者在怪企业，后者在理解企业。

### 立场自检方法（两步）

**第一步：圈词法**
圈出脚本中的主语（企业/工厂/实验室/客户/检测机构），看后面跟着什么动词：

- 高风险（你在评价客户）：「忽视、不懂、错误、违规、落后、不重视、不规范」→ ≥2个，立场有偏差
- 安全（你在理解客户）：「面临、遇到、需要、关注、希望、解决、优化」

**第二步：销售验证（最狠的方法）**
问自己：**「如果我是大客户销售，我会不会直接把这段话发给我的客户？」**
销售犹豫或觉得不合适 → 立场已经偏了。

### 第三步：行动引导检查

脚本看完后客户的感受：

- 「他说的是不是我？怎么感觉在批评我？」→ ❌ 会导致划走
- 「学到了」→ ❌ 没行动
- 「原来还有这个风险，我回头看看」→ ⚠️ 及格
- 「收藏/转发给同事」→ ⚠️ 开始有价值
- 「想咨询/想要方案/想了解设备」→ ✅ 优秀

### 第四步：短视频节奏检查

脚本是为短视频（抖音/视频号）制作的，不能写成新闻稿或文章。检查以下四个维度。

#### 1. 呼吸感——是一口气说完还是让人喘口气

核心判断：去掉所有换行符，连成一段话——听起来像不像一个人正常说话？

**人说话的自然节奏**是：
- 抛一个钩子
- 停顿，让人消化
- 再抛下一个信息
- 再停顿

❌ 反面（原版）：
```
园林绿化、林地利用、土地改良，看起来都和土地有关
但从检测标准适用上看，它们不能直接归为农用污泥
```
→ 一行塞了两个信息（种类列举+判断），呼吸感差，读起来像在念稿子。

✅ 正面（修改后）：
```
污泥用于土地，就叫农用污泥吗？
——不一定！
普通园林绿化、林地绿化、土地修复，
看着像，但从检测标准上，它们不算农用污泥！
```
→ 先抛问题 → 给意外答案 → 展开解释 → 再强调结论。每句只给一个信息点。

**检查项**：通读脚本，数一数哪些句子塞了≥2个信息点（事实+判断、定义+举例、场景+后果塞在一句里的），标记出来。

#### 2. 口语感——是人说出来的话还是写出来的文章

对比下面两段：

❌ 写出来的文章感：
```
真正意义上的农用污泥检测标准只适用于耕地、园地、牧草地三个应用场景
```
→ 句式规整，像书面定义，没有人说话时自然的节奏。

✅ 口语化的版本：
```
污泥用于土地，就叫农用污泥吗？
——不一定！
普通园林绿化、林地绿化、土地修复，
看着像，但从检测标准上，它们不算农用污泥！
```
→ 「——不一定！」像人对你说话，「看着像」像日常聊天，句子短，有语气词。

**检查口语化的三个方法：**

**方法一：朗读法** —— 把脚本默读一遍，找到那些读起来像在念稿子的句子。

**方法二：删词法** —— 删掉「针对」「进行」「相关」「方面」「基于」「通过」这类书面连接词，看句子是否还通顺。通顺→原本是书面语，删掉才像人话。

**方法三：口语替换表** —— 遇到以下词直接标红：

| 书面词 | 口语替代 |
|--------|--------|
| 针对…… | 如果是……/说到…… |
| 进行检测 | 做检测、测一下 |
| 相关要求 | 什么规定、啥标准 |
| 基于上述原因 | 所以、说白了 |
| 相应配置方案 | 怎么配、用什么设备 |

#### 3. 信息递进——是一层一层给还是一锅端

❌ 平铺一锅端：
```
农用污泥下接土壤和食物链，上游成分又不稳定，检测必须更谨慎
我们重点检测四大项：重金属含量、有机污染物风险、卫生学指标、理化指标
```
→ 因果关系（下接土壤→必须谨慎→四大项）全塞在一段里说完了，观众来不及消化。

✅ 一层一层给：
```
污泥上游成分复杂，下游连接土壤和食物链。
检测必须更谨慎——重点盯住这四大项：
重金属、有机污染物、卫生学指标、理化指标。
```
→ 先说为什么（上游复杂+下游连接土壤）→ 再说怎么办（重点盯住）→ 再具体列（四大项）。信息分三层递进，每层一句。

**检查原则**：每句话只说**一件事或一个关系**。连着读三句话，如果每句都在讲不同的东西，说明信息密度太高，需要拉开。

#### 4. 判定汇总

##### 节奏三要素

- **呼吸感**：去掉换行连起来，读一遍像不像自然说话？有没有一句话塞了≥2个信息点？
- **口语感**：有「针对」「进行」「相关」「基于」这些词吗？用朗读法读一遍，哪里像在念稿？
- **信息递进**：是事情一件一件说，还是因果/对比/并列关系全挤在一块？

##### 通用检查四问

| # | 检查项 | 具体说明 |
|---|--------|--------|
| 1 | 万能套话 | 有没有放任何行业都成立的万能句？（如「学问和门道还有很多」「不是那么简单」）→ 替换为具体画面 |
| 2 | 节奏同质化 | 段落间是不是用了同一种节奏结构？（例如五个问答段都是「问→答→展开」）→ 刻意做变化，每段换个节奏型 |
| 3 | 结论位置 | 结论是独立成句加重了，还是藏在原因后面当从句？（「要求不一样所以要做分区」vs「所以——必须功能分区」）→ 结论单独拎出来 |
| 4 | 风险可视化 | 抽象风险描述有没有画面感？（「管理和通风会更复杂」vs「管理乱、通风难、还有安全隐患」）→ 给三个具体画面词 |

##### 判定标准

- ✅ 三项节奏等要素 + 四问检查均合格 → 节奏好，可以直接录
- ⚠️ 1-2项有问题 → 标记具体位置，给修改方向
- ❌ 三项节奏要素或四问中有多项有问题 → 文案化了，需要大段重写

## 输出格式

```
📋 中实仪审稿报告

【客户画像】
🎯 目标客户：[行业+岗位]
📐 业务场景：[具体场景]
😰 核心焦虑：[具体痛点]
评级：✅ 清晰 / ⚠️ 模糊 / ❌ 缺失

【立场】
🔍 踩雷检查：
  [监管/专家/供应商/批判者] → [有/无]，[具体语句]
🧹 风险动词：[列出高风险词] 共X个

✅ 通过项：
❌ 需修改：

【行动引导】
客户感受：[会被教育 / 想回头看看 / 想咨询]
建议：[增强引导的具体方式]

【短视频节奏】
· 呼吸感：[✅ 自然 / ⚠️ 塞太多信息 / ❌ 念稿感]
· 口语感：[✅ 像人说话 / ⚠️ 少量书面词 / ❌ 像写文章]
· 信息递进：[✅ 一层层给 / ⚠️ 信息偏密 / ❌ 一锅端]

通用检查：
· 万能套话：[有/无] → 具体语句
· 节奏同质化：[有变化/同质] → 具体问题
· 结论位置：[独立/藏后面] → 建议
· 风险可视化：[有画面/抽象] → 建议

判定：[✅ 能录 / ⚠️ 标记修改 / ❌ 需重写]

【总评】
[一段话总结，指出最强点和最需要改的点]

【参考思路】
[建议的切入角度、情绪基调、内容结构、一句话标题参考]

【修改示范】
[可选：问题严重时给一段改写对比]
```
